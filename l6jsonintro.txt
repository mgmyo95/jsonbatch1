=>JSON (JavaScript Object Notation)

=>JSON 
JavaScript Object Notation 
Base on Text Format 
Light-weighted 
Does not support comments and namespace 
NoSQL database like MongoDB use JSON formats 

=> JSON 
Need to enclose name of key withing double quotes (should not use single quotes)
But can use single quotes to enclose key names 

=>JSON  Vs  XML
JSON is more consise/lightweight/easytounderstand
XML is more text longer and look more complex 

=> JSON Syntax Rules 
JSON data to written inside curly bracket 
Data is represented as key value pairs 
Keys should be enclosed in double quotes 
Key and value should be separated by colon (:)
Comma (,) is used to separated data
